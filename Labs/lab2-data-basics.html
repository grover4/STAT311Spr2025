The data

To begin exploring the functionality of R, we will work with a real dataset. Typically, analysts upload their own external dataset of interest with the Import Dataset button in the environment panel (datasets can be loaded from .txt files, .csv files, Excel files, etc.). However, many R packages have some great datasets that we can access directly once the package has been loaded.

One such dataset is called arbuthnot from the openintro package. This dataset contains the work of Dr. John Arbuthnot, an 18th century physician, writer, and mathematician. He was interested in the ratio of newborn boys to newborn girls, so he gathered the baptism records for children born in London for every year from 1629 to 1710.

To take a look at the dataset, type the following in the Console.

data(arbuthnot)  #type this in Console

The single line of code instructs R to load some data: the Arbuthnot baptism counts for boys and girls. You should see that the Environment tab in the upper right hand corner of the RStudio window now lists a data set called arbuthnot that has 82 observations on 3 variables. (You may have to click on the word “Environment” in the top right if the window is closed)

Manipulating the data

Today we will learn about four verbs from the dplyr package that we can use to wrangle our datasets. They are select, filter, mutate and arrange. Each takes a dataframe as input and does something to it. Let’s see what each one does and how to use them.

Selecting a column

We can access the data in a single column of a data frame using the command:

arbuthnot$boys   #type in Console

This command will only show the number of boys baptized each year. The dollar sign basically says “go to the data frame named arbuthnot and find the column called boys. While the dollar sign is still commonly used in R, functions in the tidyverse package allow us to extract the same information in a different way.

Type the following command in the Console and run it:

select(arbuthnot, boys)    #type in Console

The first argument is the dataframe and the second is the column we want to select. The advantage of select is that we can select several columns at once, which we cannot do with the dollar sign approach.

select(arbuthnot, boys, girls) #type in Console

You may also have noticed a subtle difference in the way that R printed the data for the previous commands. For arbuthnot$boys, R printed a list of 82 numbers that were no longer displayed as a table. Objects that print out in this way are called vectors; they represent a set of numbers.

For select(arbuthnot, boys), R printed the first 10 rows of a table with 82 rows and 1 column. This is because when you call select(arbuthnot, boys), R actually creates a new, smaller data frame. This is common to all functions in the dplyr package.

What command would you use to extract the girls? Try it in the Console.

There are a variety of helper functions that make it easy to select large numbers of columns, and you will hear more about this in class.

An important operator that we will discuss next is the pipe denoted by %>%. In short, it enables us to pipe an argument forward into a function, something along the lines of x %>% f rather than f(x).

As an example, when we type

arbuthnot %>% select(boys)    #type in Console

we are saying “take the arbuthnot dataset and supply it as input to the first argument of select function”. The main advantage of the pipe operator is that it allows us to string several commands together.

Filtering rows

Apart from select(), another useful function for subsetting a dataset is filter(). While select() lets us isolate columns of a dataset, filter() lets us isolate rows which satisfy certain attributes. Like select, the first argument is the data frame (which may supplied via the pipe operator) and the second states the criteria which must be satisfied.

What does the following line of code do?

arbuthnot %>% filter(year > 1685)  #type in Console

Try it in the Console!

The most commonly used logical operators for filtering are given below:

  • == means “if equal to”
  • != means “not equal to”
  • & means “and”
  • | means “or”
  • > means “greater than”
  • < means “less than”
  • >= means “greater than or equal to”
  • <= means “less than or equal to”

Write code to answer the questions below. Then run it and answer the questions in complete sentences immediately below each code chunk.

We will now see our first example of piping two commands together. Suppose we want to know how often more than 5,000 girls were baptized. We first use filter to extract the rows that have girls \(>\) 5000. Then we go ahead and count the number of rows using nrow. This is shown below. Try it.

arbuthnot %>% filter(girls > 5000) %>% nrow()  #type in Console

For ease of reading, you may break up the code over several lines so each task is on a separate line.

arbuthnot %>% 
   filter( girls > 5000 ) %>%
   nrow()

When breaking up lines of code however, be careful to make sure that each line is syntactically incomplete, so R knows that there is more coming. For example, placing the %>% at the end of a line rather than the beginning of the next line does the trick.

Mutate new columns

The third verb we will learn about today is mutate. It enables us to create new columns in our dataset. For example, suppose we want to know the total number of baptisms in a year. To compute this, we could use the fact that R is really just a big calculator. We can type in mathematical expressions like

5218 + 4683

to see the total number of baptisms in 1629. We could repeat this once for each year, but there is a faster way. We can have R create a new variable for us, and we can define this new variable with a mathematical expression. Try out the following code in the Console:

arbuthnot %>% mutate(total = boys + girls)  #type in Console

This should print the desired dataframe (arbuthnot with the new total column). Click on the name of the dataset in your environment again to update it in the data viewer. Has the column total been added to the dataset?

To add the new column to the dataset, you will need to assign the output from the previous command using the assignment operator <-.

arbuthnot <- arbuthnot %>%     #type in Console
                mutate( total = boys + girls)

You should see that there is now a new column called total that has been tacked on to the data frame. You have changed your dataset using mutate!

Finally, in addition to simple mathematical operators like subtraction and division, we often create new variables by using logical operators like greater than, >, less than, <, and equality, == and even conditional ifelse statements. For example, we can ask if the number of baptisms of boys outnumber that of girls and create a new variable which takes the value “yes” or “no” depending on the answer.

arbuthnot <- arbuthnot %>%     #type in Console
  mutate(more_boys = ifelse(boys > girls, "yes", "no"))

Arrange in order

A large part of initial data exploration is examining the values taken by relevant variables. The verb arrange orders the rows of a data frame in increasing order by the values of selected columns.

For example, say we want to know the year with the most female baptisms. Conceptually, we would first sort your data set in decreasing order based on the girls column. You can do this interactively in the data viewer by clicking on the arrows next to the variable names. To include the sorted result in your report however you will need to use two new functions. First we use arrange() to sort the girlsvariable. Then we can arrange the data in a descending order with another function, desc(), for descending order. The sample code is provided below. Try it in the Console.

arbuthnot %>%
  arrange(desc(girls))   # type in Console
